# UHB ↔ BHSA Strong's reconciliation — full OT

Generated by `spine/reconcile.py`. Joins the UHB content-Strong's sequence
to BHSA's (Strong's from the OpenHebrewBible `002` crosswalk) per book and
catalogues the residual.

## Result

| | matched / total | coherence |
|---|---|---|
| Raw (no equivalences) | 294,200 / 297,689 | **98.83%** |
| + 9-entry equivalence table | 296,481 / 297,689 | **99.59%** |

Every book lands 98.5–99.6% (`per_book.csv`). With the equivalence table
the residual drops from 2,926 → **645 words** across 335 distinct Strong's
— the genuine sparse tail (mostly 1–2 occurrences each).

## What the equivalence table fixed

The raw residual was dominated by a *handful* of high-frequency Strong's
variant/homograph pairs (`residual_pairs.csv`), all linguistically real:

| variant → canonical | word | occ |
|---|---|---|
| H3212 → H1980 | yalak/halak "go" | 1026 |
| H3069 → H3068 | YHWH (qere variant) | ~420 |
| H2009 → H2005 | hinneh/hen "behold" | 215 |
| H0854 → H0853 | 'et "with" / DOM | ~340 |
| H0582 → H0376 | 'enosh/'ish "man" | ~190 |
| (+ Egyptian, bachur, 'al, samach) | | ~60 |

These ~9 entries closed ~78% of the residual. See
`spine/strongs_equivalence.tsv`.

## Files

- `per_book.csv` — per-book match rate (book, uhb_strongs, bhsa_strongs, matched, pct)
- `residual_strongs.csv` — unmatched BHSA Strong's by frequency
- `residual_pairs.csv` — substitution pairs (uhb_strong, bhsa_strong, count) → equivalence-table candidates

## Caveats

- Strong's per BHSA word comes from the `002` crosswalk (keyed to BHSA
  **etcbc4c**); books are sliced by BHSA-2021 word counts, so a few words
  of version drift land in the residual. For a per-node production map,
  align the crosswalk to the target BHSA version by reference + surface.
- The last ~0.4% (645 words) is sparse; chasing it to 100% is diminishing
  returns and not needed for the loose join.
